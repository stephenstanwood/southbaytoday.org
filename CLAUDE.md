# southbaytoday.org

## Architecture
- Astro 6 + Vercel + React + Tailwind v4
- Standalone app broken out from stanwood.dev
- Single page (src/pages/index.astro) with tab-based navigation
- All SBS components in src/components/south-bay/
- Static data in src/data/south-bay/ (generated by scripts/)
- Shared lib in src/lib/south-bay/

## Data Generation
Run these scripts to refresh pre-generated data:
- `npm run generate-digests` — council meeting summaries (needs ANTHROPIC_API_KEY)
- `npm run generate-events` — upcoming events from 20+ sources
- See `package.json` for full list of `generate-*` scripts

## File Organization
- Generated JSON artifacts (committed): `src/data/south-bay/*.json`
- Runtime social state (gitignored): `social-*.json` files in same dir
- Central path config: `scripts/lib/paths.mjs`

## Deployment
- Vercel project connected to this repo
- Custom domain: southbaytoday.org
- stanwood.dev/south-bay redirects here permanently
- NEVER create files in a root `api/` directory — Vercel treats that as legacy serverless functions and routes all `/api/*` traffic away from Astro's `_render`, breaking every API route. All API routes must live in `src/pages/api/`.

## Events tab top is curated — no banners
**Guiding principle:** the search bar, filters, and event list must be reachable WITHOUT scrolling on a typical desktop or mobile viewport. Don't add anything above the filter bar (`sb-events-sticky-filter`) that pushes it below the fold.

Do NOT add heads-up banners, observance strips, school-calendar panels, holiday call-outs, or any other "above the filter bar" sections to `src/components/south-bay/views/EventsView.tsx`. Five banners were removed 2026-05-13 — SchoolHeadsUpBanner, SchoolYearEndgamePanel, HolidayHeadsUpBanner, HolidayPicksPreview, HeritageMonthBanner — because they pushed the filter bar off-screen (especially the school-dates content, which gets re-added on autopilot). The data files (`school-calendar.json`, `holidays.ts`, `heritageMonths.ts`) stay because they're used elsewhere — that's NOT a signal to re-wire a banner.

`WeekendPicksStrip` is the ONLY top-of-Events strip Stephen has approved. It's conditional (Fri–Sun only) and compact. Don't add siblings to it. Routine work on filters, date pills, event cards, etc. is fine.

## Don't surface "this was built but never rendered" components
When you find an orphan component file or a data file with no visible UI, that is almost always intentional — either Stephen removed the rendering or the data is used elsewhere (social pipeline, day-plan generation, freshness checks). Do NOT "complete" it by wiring it in. Surface a 1-line note in your end-of-turn summary if you think something is genuinely missing, and let Stephen decide.

## Newsletter — every item gets a CRAFTED image, never a generic icon
Every newsletter row (events, openings, conversation/"Around the South Bay") should carry a real or **per-item crafted** image. The guarantee is a Recraft backstop, NOT a fallback tile:
- `scripts/newsletter/ensure-images.mjs` (`ensureNewsletterImages`) runs in `send.mjs` after assemble / before render (real sends only — skipped for `--test`/`--dry-run`). It Recrafts a per-item tile for anything still imageless after the generators, uploads to Blob, and caches the URL in the **committed** `newsletter-image-cache.json` keyed by item fingerprint so it's never re-billed.
- This is the backstop half; the upstream generators (`generate-events` Tier-4, `generate-scc-food-openings`, `reddit-pulse`) still craft what they can at ingest.
- Do **NOT** add a generic branded fallback icon (a one-size logo tile looks worse than nothing). `icon-512.png`/`logo-mark.png` as a newsletter image fallback was tried and removed.
- In `lib.mjs`, rows use `squareThumbImg(url, size, radius)`: it returns "" when there's genuinely no usable image, and the block **drops the whole image cell** so the row renders text-only at full width — never an empty left-column gap, never a generic icon. The crafted image is also set as the `<img>` background so a client that fails the `<img>` still shows the art. Don't reintroduce a static fallback or an always-present empty cell.
